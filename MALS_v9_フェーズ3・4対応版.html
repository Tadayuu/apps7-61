Firebase リアルタイム同期システム

多端末での同期を行う場合は、下記の項目をFirebase Consoleの値でひとつずつ入力してください（未設定時はブラウザのローカルストレージのみで動作します）。

  1. Firebase Console（console.firebase.google.com）でプロジェクトを開く
  2. 左上の歯車アイコン（）から「プロジェクトの設定」を開く
  3. 「全般」タブ内の「マイアプリ」で、ウェブアプリ（</>アイコン）の「SDK の設定と構成」を開く
  4. 表示されたコードの中の値を、下の各項目にひとつずつコピーする（下の自動入力欄に丸ごと貼り付けてもOK）
  5. データベースURLだけは「構築」メニューの「Realtime Database」画面の上部に表示されるURLを使う（Firestoreの画面ではないので注意）

【フェーズ1】Firebase セキュリティルール設定ガイド

このアプリの「管理者ロック」はクライアント側のUIガードであり、Firebase Realtime Database 自体への不正な読み書きを防ぐものではありません。 本番運用する場合は、必ず Firebase Console の「Realtime Database > ルール」で下記のようなアクセス制御を設定してください（未設定のままだと誰でもデータベースを閲覧・書き換えできる状態になります）。 

{
  "rules": {
    "mals_v6_system": {
      ".read": "auth != null",
      ".write": "auth != null"
    }
  }
}
コピーしました

上記ルールは Firebase Authentication の導入を前提としています。認証基盤を未導入のまま厳格化したい場合は、一時的に固定のシークレットキーを条件式に組み込む方法もありますが、恒久対応としては Firebase Authentication への移行を推奨します（本仕様書フェーズ3で計画）。

アダプティブ問題エディタ

KaTeX記述（$数式$）に対応。レベル別の問題をプールに補填します。

空白・全角半角の違いを無視して判定します。表記ゆれが多い場合は複数行に列挙してください。

図形描画問題は自動採点ができないため、生徒がキャンバスに描画→模範解答を確認→自己申告で正誤を記録する方式です。

小単元とその配下の問題を「テンプレート」として書き出すと、他のクラスを担当する教員も同じFirebaseプロジェクト上でインポートして再利用できます。

共有ライブラリ一覧（全クラス共通・登録した教員・クラス名を表示）

【フェーズ4】PDF / Word からの問題自動抽出インポート（ベータ）

既存のプリントやWord問題集からテキストを抽出し、パターンに一致する箇所を問題候補として自動検出します。抽出はブラウザ内で完結し、外部サーバーには送信されません。
対応フォーマットの目安：「問1．」「(1)」などの通し番号＋本文、「①②③④」または「1.2.3.4.」形式の選択肢、「正解：②」のような解答表記。書式が異なる場合は検出精度が下がるため、必ずインポート前のプレビューで内容を確認・修正してください。
